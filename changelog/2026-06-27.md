- Java Agent无侵入式接入，无需修改代码
- 自动埋点覆盖：HTTP请求、SQL执行、Redis命令
- 支持在Grafana中查看完整调用链和Trace详情

#### Spring Retry 重试框架
- 集成Spring Retry + Spring AOP
- `incrementViewCount` 方法示例：3次重试，指数退避（100ms起步，2倍递增）

#### 业务自定义指标
- `campushare.post.create.total`：发帖量（按学校、帖子类型分类）
- `campushare.post.create.duration`：发帖延迟
- `campushare.file.upload.total`：文件上传次数
- `campushare.file.upload.size`：文件大小分布
- `campushare.file.upload.failed`：上传失败次数

### Bug修复：文件上传问题

#### 问题1：FileNotFoundException（No such file or directory）
- **根因**：上传路径 `./uploads/` 为相对路径，在Docker容器中被解析到Tomcat临时工作目录，容器重启后文件丢失
- **修复**：
  - `application-docker.yml` 配置绝对路径 `/app/uploads/`
  - `FileController` 增加 `@PostConstruct` 初始化方法，服务启动时自动创建上传目录
  - 使用 `Paths.get()` 替代字符串拼接，避免跨平台路径分隔符问题
  - docker-compose.yml 新增 `uploads_data` 数据卷，挂载到 `/app/uploads`，确保持久化

#### 问题2：大文件上传413错误
- **根因**：Nginx默认 `client_max_body_size` 为1MB，网关Netty默认请求体大小限制
- **修复**：
  - 前端Nginx配置 `client_max_body_size 100M`
  - 网关Netty配置 `max-content-length: 104857600`（100MB）

### 技术细节
- docker-compose.yml新增prometheus、grafana、tempo服务及对应数据卷
- 所有容器统一配置 `TZ=Asia/Shanghai` 时区

## [2026-06-27] 浏览量/点赞/收藏/个人主页全链路重构

> 本次为大型功能重构，涉及后端核心互动逻辑、个人主页体系、数据初始化、网关安全、前端多页面改造。

### 核心问题与修复

#### 问题1：收藏/点赞只存Redis不写DB，数据7天后丢失
- **根因**：`toggleStar()`/`toggleLike()` 只在Redis设置键（7天TTL），从未写入 `post_stars`/`post_likes` 表
- **修复**：DB为数据源 + Redis为缓存。toggle操作先写DB（insert/delete），再同步Redis缓存（30天TTL）；计数改为 `setSql("star_count = star_count + 1")` 原子操作

#### 问题2：浏览量不记录历史，无法联动个人主页
- **根因**：`incrementViewCount()` 只做Redis计数累加，不记录 `view_history` 表，且不接收userId
- **修复**：方法签名改为 `incrementViewCount(userId, postId)`；直接DB原子+1（不再用Redis batching）；在 `view_history` 表upsert记录（同用户同帖子只保留一条，更新最近浏览时间）

#### 问题3：帖子详情页收藏/点赞状态不持久化
- **根因**：前端完全用mock数据，点击只改本地state不调接口
- **修复**：后端新增 `GET /api/posts/{postId}/status` 接口返回 `{starred, liked}` 状态；前端进入时从后端获取详情+状态，点击调用真实接口，带乐观更新+失败回滚

#### 问题4：列表页收藏按钮只变色不调接口
- **根因**：handleStar只操作本地Set state，没调用后端接口
- **修复**：改为async调用 `postApi.toggleStar`，乐观更新+回滚；fetchPosts加载列表后批量查询每个帖子的真实状态

#### 问题5：个人主页三个Tab全用mock数据
- **根因**：浏览历史/收藏/点赞用硬编码mock数据，Tab形式内嵌展示
- **修复**：改为抖音风格入口按钮，点击跳转独立列表页；新建 `MyListPage.tsx` 通用列表页，根据路由参数调用对应后端接口；路由合并为 `/profile/:type` 一条；增加 `location.key` 依赖，从详情页返回时自动重新拉取数据

#### 问题6：缺少个人主页接口
- **修复**：新增5个接口：
  - `GET /api/posts/history` — 浏览历史，按view_time倒序
  - `GET /api/posts/starred` — 我的收藏，按收藏时间倒序
  - `GET /api/posts/liked` — 我的点赞，按点赞时间倒序
  - `GET /api/posts/mine` — 我的帖子，按发布时间倒序
  - `GET /api/posts/my-stats` — 我的帖子统计数据（总浏览/获赞/被收藏/帖子数）

#### 问题7：网关白名单过宽
- **根因**：白名单含 `/api/posts/`，用 `path::contains` 匹配导致所有 `/api/posts/**` 绕过JWT
- **修复**：移除 `/api/posts/`，仅保留 `/api/posts/school` 和 `/api/files/` 公开

#### 问题8：数据初始化逻辑不符合调试需求
- **根因**：生成随机计数（0-5000）、学校ID1-12（init.sql只有1-8）、含note类型、无清空功能
- **修复**：
  - 新增 `POST /api/admin/clear-posts` 接口，物理删除posts/post_stars/post_likes/view_history + 清Redis
  - `initTestData()` 修改：计数全0、学校1-8、仅resource/discussion、默认10条/校
  - 测试虚拟文件大小改为1KB，避免OOM；移除 `@Transactional`，改用setter替代builder确保MyBatis Plus自动填充生效；增加per-insert try-catch

## 快速参考：调试数据接口

项目内置测试数据管理接口，通过网关（8080端口）调用，**无需登录**（已在网关白名单 `/api/admin/`）：

| 接口 | 方法 | 参数 | 说明 |
|------|------|------|------|
| `/api/admin/clear-posts` | POST | 无 | 清空所有帖子及关联数据 + Redis缓存 |
| `/api/admin/init-test-data` | POST | `postsPerSchool`（默认10） | 为学校1-8生成测试帖子，计数全0 |

**调试时重置数据流程**：
```bash
# 1. 先清空旧数据
curl -X POST http://localhost:8080/api/admin/clear-posts

# 2. 重新生成（每校10条）
curl -X POST "http://localhost:8080/api/admin/init-test-data?postsPerSchool=10"
```
