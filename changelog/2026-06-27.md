## [2026-07-01] 修复 Agent 切换导航栏丢失对话状态 + 新增架构总览文档

### Bug修复
- **切换导航栏后 Agent 对话丢失**：根因是 `currentSessionId` 和 `messages` 使用 `useState` 管理，React Router 切换路由时组件卸载导致状态丢失，重新挂载时总是初始化为欢迎消息和 null sessionId。修复方案：用 `localStorage` 持久化 `currentSessionId`（key: `agent_current_session_id`），组件挂载时自动读取并调用 `loadSession` 恢复上次对话；仅用户主动点击「新建对话」才清除持久化并重置；会话被删除导致加载失败时自动回退到欢迎界面

### 新增文档
- `docs/agent-assistant/architecture-overview.md`：Agent 模块架构总览导航文档（1059 行），含 14 章节 12 张 Mermaid 图，覆盖全景架构、核心概念关系（RAG/MCP/Tool/Function Calling/Skill/记忆系统/上下文工程/提示词工程）、问答完整流程、ReAct 循环、上下文 L0-L5 分层、记忆系统架构、SSE 事件流、技术栈全景表、演进路线、部署拓扑、当前实现状态

### 改动文件
- `frontend/src/pages/AgentPage.tsx`（会话状态持久化）
- `docs/agent-assistant/architecture-overview.md`（新增架构总览）

## [2026-07-01] 修复 Agent 智能助手前端布局与后端 Docker 连接问题

### Bug修复
- **对话框被底部导航栏遮挡**：AgentPage 最外层 div 添加 `pb-16`（64px），为 fixed 定位的 NavBar 预留底部空间
- **对话失败返回"服务异常"**：根因是 agent-service 缺少 `application-docker.yml`，Docker 容器内用 localhost 连 MySQL/Redis 全部失败。新建 application-docker.yml，将地址改为 Docker 服务名（mysql、redis、agent-postgres），用户名密码与其他服务一致（root/root123456）
- **对话内容无法选中复制**：消息气泡添加 `select-text` 类，确保文本可选择
- **agent-service 缺少 DNS 配置**：docker-compose.yml 中 agent-service 添加 `dns: [8.8.8.8, 114.114.114.114]`，确保容器内能解析外网 api.deepseek.com 域名（与 user-service 保持一致）

- `frontend/src/pages/AgentPage.tsx`（布局修复 + select-text）
