# Changelog 2026-07-01

## Agent 智能助手 Week 2 RAG 实施（模块 1-11 完整落地）

### 业务背景（主动需求型 - 用户驱动）

**Why now**: 用户反馈 agent 智能助手「小享」回答项目功能时存在 LLM 幻觉——例如告诉用户"注册登录分为手机和邮箱"，但实际 LoginRequest.java 明确支持用户名/手机号/邮箱三种登录方式。根因是 system prompt 只描述泛泛功能名，模型靠常识补全细节，无真实依据。

**What if not done**: 用户按 agent 的错误指引操作会失败（如尝试用用户名登录却发现 agent 说只支持手机/邮箱），严重损害产品信任度。agent 作为帮助中心的核心价值丧失。

**What success means**: agent 回答"怎么登录注册"时准确引用知识库（用户名/手机号/邮箱三种登录 + 手机/邮箱两种注册），回答"怎么发帖"时准确说明 resource/discussion 两种类型和文件上传流程，所有功能性问题基于项目真实代码回答。

### 问题

1. **LLM 幻觉**：system prompt 只写功能名，模型凭常识补全细节
2. **无知识检索**：agent 不掌握项目真实代码和功能细节
3. **帖子不可检索**：用户发布的帖子内容无法被 agent 检索引用

### 解决方案

**模块 1-8（知识库 RAG 核心）**：
- 切换 embedding 服务到硅基流动 BGE-M3（1024 维稠密向量）
- PgVectorConfig 双数据源隔离（MySQL 业务表 + PostgreSQL/pgvector 向量库）
- EmbeddingClient + CircuitBreaker + Retry 保护
- KnowledgeVectorStore / PostVectorStore（向量相似度 + pg_trgm 关键词混合检索）
- 18 篇知识库文档（从真实代码提取：登录/注册/发帖/点赞/收藏/评论/文件上传/分类/角色等）
- KnowledgeIngestionService（MD5 增量检查，启动 30s 首次摄入，每小时检查）
- RetrievalService（三路检索：知识库向量 + 知识库关键词 + 帖子向量，RRF 融合 k=60）
- PromptConstants 新增 {{RETRIEVAL_CONTEXT}} 占位符，AgentChatService 注入检索上下文

**模块 9（post-service 帖子向量通知）**：
- AgentFeignClient（post-service → agent-service 异步通知）
- InternalPostController 新增 /all-for-vector 和 /{postId}/vector-data 端点
- PostServiceImpl createPost/editPost/deletePost 异步通知（CompletableFuture.runAsync + try-catch）
- 配置 service.agent.url

**模块 10（agent-service 帖子向量服务）**：
- PostFeignClient（agent-service → post-service 拉取帖子数据）
- PostVectorService（syncPost 单帖同步 + syncAll 分页全量同步，批量 embedding batch-size=32）
- InternalAgentController 新增 /posts/sync 和 /posts/reindex 端点
- PostVectorScheduler（启动 60s 首次全量同步，每 5 分钟兜底全量同步）
- postSyncCircuitBreaker 独立熔断（post-service 故障不影响 agent 对话）

**模块 11（验证部署）**：
- 本地 JDK 17 编译验证通过（campushare-agent + campushare-post）
- git commit 3fe25bc 推送至 origin/master（51 文件，2765 行）

### 效果

- **幻觉消除**：agent 回答"怎么登录注册"基于知识库 02-login-guide.md，准确包含用户名/手机号/邮箱三种登录方式
- **混合检索**：知识库 18 篇 HOW_TO 文档 + 用户帖子内容，RRF 融合后 top-5 注入 prompt
- **降级策略**：embedding 失败 → 空检索上下文 → 对话不中断；post-service 不可用 → 帖子向量不更新 → 知识库检索仍正常
- **兜底机制**：PostVectorScheduler 每 5 分钟全量同步，即使通知丢失也能保证向量库一致性
