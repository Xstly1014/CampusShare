### 架构：三层漏斗 + 5 大意图 + 14 子意图

**5 大 L1 意图**：
| 意图 | 标签 | 说明 |
|------|------|------|
| HOW_TO | 操作指引 | 用户询问怎么使用功能、如何操作 |
| SEARCH | 内容检索 | 用户想找资料、求资源 |
| NAVIGATE | 页面导航 | 用户想知道某个功能在哪、入口在哪 |
| CLARIFY | 多轮澄清 | 指代词、追问、修正上一轮 |
| OUT_OF_SCOPE | 超范围 | 闲聊、写操作、敏感话题 |

**14 个 L2 子意图**：
- HOW_TO: feature_help / rule_explain
- SEARCH: resource / discussion / content_qa
- NAVIGATE: feature_loc / section_loc / my_list
- CLARIFY: coreference / refine / followup
- OUT_OF_SCOPE: chitchat / open_domain / write_action / sensitive

**三层漏斗架构**：
```
用户 query
  ↓
Layer 1: RuleShortCircuitFilter（规则短路，<5ms）
  ├─ 命中 → 返回 IntentResult (confidence≥0.9, layer=RULE)
  └─ 未命中 ↓
Layer 2: IntentClassifier（LLM 分类，~300ms）
  ├─ 成功 → 返回 IntentResult (layer=LLM)
  └─ 失败/熔断/超时 ↓
Layer 3: EmbeddingIntentFallback（向量相似度，~80ms）
  ├─ 成功 → 返回 IntentResult (layer=EMBEDDING)
  └─ 失败 ↓
Default: 兜底 SEARCH (layer=DEFAULT, confidence=0.0)
```

### 关键设计决策（ADR）

- **ADR-009**：子意图用 String 常量而非枚举，便于扩展不破坏现有代码
- **ADR-010**：置信度 < 0.6 视为低置信，兜底为 SEARCH（边界值 0.6 属于高置信）
- **ADR-011**：分类 + 查询改写 + 槽位抽取合并为一次 LLM 调用，省 ~500ms 延迟和 1 次 API 成本
- **ADR-013**：IntentRouter 快路径 — OUT_OF_SCOPE/NAVIGATE 走模板回复（0 LLM），HOW_TO/SEARCH/CLARIFY 走 RAG 管线
- **ADR-014**：Redis 缓存相同 query 的意图分类结果，TTL=1h，key=`agent:intent:{md5(query)}`

### IntentRouter 快路径

| 意图 | 子意图 | 行为 |
|------|--------|------|
| OUT_OF_SCOPE | chitchat | 模板：自我介绍 + 引导 |

### 修复过程

1. **Mono.zip 编译错误**：Reactor 的 `Mono.zip()` 无 `List<Mono<T>>` 重载 → 改用 `Flux.fromIterable().flatMap().collectList()`
2. **void 方法 mock 失败**：`ValueOperations.set()` 返回 void，Mockito `when().thenThrow()` 仅适用于 non-void → 改用 `doThrow().when()`
3. **Jackson 反序列化失败**：IntentResult 缺少 `@NoArgsConstructor`，Jackson 无法构造 → 添加 `@NoArgsConstructor` + `@AllArgsConstructor`
4. **isHighConfidence/isLowConfidence 被误识别为 Jackson getter**：`isXxx()` 方法被 Jackson 序列化为 `highConfidence`/`lowConfidence` 属性，反序列化时找不到 setter 抛异常 → 添加 `@JsonIgnore`
5. **SlotResult 反序列化**：内部类缺少 `@NoArgsConstructor` → 防御性添加（缓存含 slots 的 IntentResult 时需要）
6. **Markdown 代码块解析**：测试用 `\\n`（字面量反斜杠+n）而非 `\n`（真实换行符）→ 修正为 `\n`
7. **emptyChoices 测试未 stub embeddingFallback**：Mockito 默认返回 null → 补充 `when(embeddingFallback.classify()).thenReturn(Mono.just(...))`
8. **IntentRouter 路由匹配顺序**：`Map.of()` 无序，"我点赞的帖子" 同时匹配 "点赞" 和 "帖子"，"帖子" 可能先匹配 → 改用 `LinkedHashMap`，"帖子" 放最后

### 架构：分块 embedding + 四维质量评分 + 双阈值重复检测 + SemVer 版本管理

**分块 embedding**：
```
知识文档 → MarkdownChunker（H2→H3→段落→句子，256 Token 目标，512 Token 最大，50 Token 重叠）
  → EmbeddingClient.embedBatch（BGE-M3，1024 维）
  → KnowledgeVectorStore.upsertChunks（PG knowledge_vectors 分块表）
```

**四维质量评分**：
```
finalScore = rrfScore * (0.8 + 0.2 * qualityScore)
qualityScore = recall × 0.4 + feedback × 0.3 + freshness × 0.2 + completeness × 0.1
```

**双阈值重复检测**：
- similarity ≥ 0.95 → DUPLICATE（跳过摄入）
- similarity ≥ 0.85 → SIMILAR（warn 但仍摄入）
- similarity < 0.85 → UNIQUE（正常摄入）
- 仅查 chunk_index=0 代表性分块，异常降级为 UNIQUE

**SemVer 版本管理**：
- 版本号：vMAJOR.MINOR.PATCH（如 v1.0.0）
- 更新：nextPatch 递增（v1.0.0 → v1.0.1）
- 完整快照历史表（非 diff），回滚 O(1) 读取
