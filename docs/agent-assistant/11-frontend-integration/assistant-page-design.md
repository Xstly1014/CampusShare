# 助手主页面 UI 设计

> 状态: 草稿
> 最后更新: 2026-06-30

## 一、页面布局

AssistantPage 采用全屏聊天布局，顶部 Header + 中间消息流 + 底部输入区：

```
┌─────────────────────────────┐
│ Header: 标题 + 历史 + 清空  │  56px
├─────────────────────────────┤
│                             │
│  消息流（可滚动）           │  flex-1
│                             │
│  - 欢迎卡（首轮）           │
│  - 用户气泡（右）           │
│  - 助手气泡（左）+引用      │
│  - 工具调用状态条           │
│  - 澄清问题卡               │
│                             │
├─────────────────────────────┤
│ 输入区：输入框 + 发送 + 停止│  64px
└─────────────────────────────┘
```

不显示 NavBar（聊天页全屏沉浸），通过 Header 左上角返回按钮回到上一页。

## 二、Header 设计

```
[←]  AI 助手        [📋 历史] [🗑 清空]
```

- 左：返回按钮（`ArrowLeft` 图标），返回上一页（不是 /home，是浏览器历史后退）。
- 中：标题"AI 助手"，蓝色 #2563EB。
- 右：历史按钮（`History` 图标）→ 弹出历史会话列表抽屉；清空按钮（`Trash2` 图标）→ 弹确认弹窗"清空当前会话？"。

## 三、消息流

### 3.1 欢迎卡（首轮）

```
┌─────────────────────────────┐
│ 👋 你好，我是校园助手       │
│                             │
│ 我可以帮你：                │
│ • 查找学习资源              │
│ • 解答平台使用问题          │
│ • 定位感兴趣的板块          │
│                             │
│ 快捷提问：                  │
│ [怎么关闭通知] [找考研真题] │
│ [怎么成为创作者]            │
└─────────────────────────────┘
```

- 快捷提问是按钮，点击直接发送预设问题。
- 个性化：若 user_memory 有"主要兴趣分类=软件"，快捷提问改为软件相关问题。

### 3.2 用户气泡

```
                              ┌─────────────────┐
                              │ 怎么关闭通知？   │
                              └─────────────────┘
```

- 右对齐，蓝色背景 #2563EB，白色文字。
- 圆角 `rounded-2xl rounded-tr-sm`（右上角无圆角，指向用户）。
- 最大宽度 75%，长文自动换行。

### 3.3 助手气泡

```
┌─────────────────────────────┐
│ 关闭通知的步骤如下：        │
│                             │
│ 1. 进入「我的」→「设置」    │
│ 2. 选择「通知偏好设置」[1]  │
│ 3. 关闭对应通知开关 [2]     │
│                             │
│ 📌 关闭后仍可在通知页查看   │
└─────────────────────────────┘
[1] 通知偏好设置文档  [2] 通知中心说明
```

- 左对齐，白色背景，灰色边框。
- 圆角 `rounded-2xl rounded-tl-sm`。
- 引用 `[n]` 渲染为蓝色上标角标，可点击。
- 回答下方渲染引用卡片列表，点击跳转对应资源。

### 3.4 工具调用状态条

Agent 调用工具时，在助手气泡上方显示状态条：

```
┌─────────────────────────────┐
│ 🔍 正在搜索帖子...          │
│ ✓ search_posts (320ms)      │
│   找到 3 篇相关帖子         │
└─────────────────────────────┘
```

- 灰色背景，小字号。
- 实时更新：start → end。
- 多个工具调用堆叠显示。
- 折叠：默认只显示最近 1 条，点击展开全部。

### 3.5 澄清问题卡

当 Agent 发起 CLARIFY 时：

```
┌─────────────────────────────┐
│ 🤔 你是想关闭哪种通知？     │
│                             │
│ [点赞通知] [评论通知]       │
│ [私信通知] [全部关闭]       │
│                             │
│ 或输入其它回答...           │
└─────────────────────────────┘
```

- 选项按钮点击即发送该选项作为用户消息。
- 输入框仍可用，用户可手动输入其它回答。

### 3.6 流式生成中

LLM token 流到达时，助手气泡实时增长：

```
┌─────────────────────────────┐
│ 关闭通知的步骤如下：        │
│ 1. 进入「我的」→「设置」    │
│ 2. 选择「通知偏好设置」█    │  ← 闪烁光标
└─────────────────────────────┘
```

- 末尾闪烁光标 `█` 表示生成中。
- 引用 `[n]` 在 token 流中先显示为纯文本，refs 事件到达后转为可点击角标。

### 3.7 错误气泡

```
┌─────────────────────────────┐
│ ⚠️ 助手响应超时             │
│                             │
│ [已生成的部分回答...]       │
│                             │
│ [🔄 重试]                   │
└─────────────────────────────┘
```

- 黄色边框提示错误。
- 显示 partial_answer（若有）。
- 重试按钮重新发送同一条消息。

## 四、输入区

```
┌─────────────────────────────────┐
│ [📝] 输入问题...            [➤] │
└─────────────────────────────────┘
```

- 文本域，单行高度 40px，多行自动增高（最多 4 行）。
- 左侧附件按钮（暂不启用，预留）。
- 右侧发送按钮（蓝色圆形）。
- 生成中发送按钮变为停止按钮（`Square` 图标）。
- Enter 发送，Shift+Enter 换行。
- 空内容禁用发送按钮。

## 五、历史会话抽屉

点击 Header 历史按钮，从右侧滑出抽屉：

```
┌─────────────────────────┐
│ 历史会话            [×] │
├─────────────────────────┤
│ 📌 关于通知设置  5轮     │
│    2026-06-30 10:00     │
├─────────────────────────┤
│ 找考研真题      8轮      │
│    2026-06-29 15:30     │
├─────────────────────────┤
│ 创作者认证流程  3轮      │
│    2026-06-28 20:10     │
└─────────────────────────┘
```

- 列表展示 ARCHIVED 会话（标题/轮次/时间）。
- 点击会话：新建会话并预填"继续之前的对话：{摘要}"，不复活旧会话。
- 下拉刷新加载更多。

## 八、引用渲染细节

### 8.1 角标样式

正文中 `[1]` 渲染为：
```html
<sup class="text-blue-600 cursor-pointer hover:underline">[1]</sup>
```

### 8.2 引用卡列表

回答气泡下方：

```html
<div class="mt-2 space-y-1">
  <a class="block text-xs text-blue-600 hover:underline">
    [1] 通知偏好设置文档
  </a>
  <a class="block text-xs text-blue-600 hover:underline">
    [2] 通知中心说明
  </a>
</div>
```

### 8.3 点击行为

- post 引用 → 跳转 `/post/{postId}`（或 `/school/{schoolId}/post/{postId}`）。
- knowledge 引用 → 跳转 `/help/{articleId}`。
- user 引用 → 跳转 `/user/{userId}`。
- category 引用 → 跳转 `/category/{categoryId}`。

## 九、反馈机制

每条助手回答下方有 👍/👎 按钮：

```
┌─────────────────────────────┐
│ 助手回答内容...             │
└─────────────────────────────┘
👍 👎                        复制
```

- 点击 👍：发送 feedback=UP，按钮变蓝。
- 点击 👎：弹出预设原因（"答非所问"/"信息过时"/"太啰嗦"/"其它"），选择后发送 feedback=DOWN + reason。
- 复制按钮：复制回答纯文本到剪贴板，Toast 提示"已复制"。

## 十、决策记录 (ADR)

### ADR-106: 全屏聊天布局而非嵌入卡片
- **理由**：聊天交互需要足够垂直空间，嵌入卡片会挤压首页内容。全屏沉浸更专注。
- **代价**：失去底部 NavBar 快速切换，通过 Header 返回按钮补偿。

### ADR-107: Header 提供历史与清空而非汉堡菜单
- **理由**：校园用户对汉堡菜单认知度低，显式按钮更直观。
- **历史**：抽屉式展示，不离开当前会话。

### ADR-108: 引用角标 + 卡片双展示
- **理由**：角标便于正文中定位，卡片便于回答后查看全部引用。两者互补。
- **替代**：仅角标（不直观）、仅卡片（正文无锚点）。

### ADR-109: 流式光标 + token 实时拼接
- **理由**：让用户感知"Agent 在说话"，比转圈等待体验好 10×。
- **实现**：refs 事件到达后用正则把 `[n]` 替换为 `<sup>` 角标。

### ADR-110: 👎 弹出预设原因
- **理由**：纯 👎 无 actionable 信息，预设原因让用户低成本给出具体反馈，便于离线分析改进。
- **预设**：答非所问/信息过时/太啰嗦/其它（"其它"允许自由输入）。

### ADR-111: 不复活旧会话而是新建并预填摘要
- **理由**：旧会话上下文已归档，复活需重新装载 Redis 状态，复杂且易错。新建会话预填摘要让 LLM 快速恢复上下文，更简单可靠。
- **代价**：用户感觉"对话被总结过"，但可接受。

### ADR-112: Enter 发送 Shift+Enter 换行
- **理由**：聊天场景 Enter 发送是行业标准（微信/Telegram/Slack 都这样），用户无学习成本。
- **多行**：Shift+Enter 换行满足长输入需求。
