# 反思与自校验

> 状态: 草稿  
> 最后更新: 2026-06-30

## 一、为什么需要反思

LLM 会：幻觉引用、答非所问、超范围、遗漏关键信息。反思是在输出前自检，把这些问题拦截在用户看到之前。

## 二、反思检查项

| 检查 | 说明 | 失败处理 |
|------|------|----------|
| Grounding | 每条客观陈述是否有 refs 支撑 | 删除无支撑陈述 or 补检索 |
| 切题 | 是否回答了用户问题 | 重新生成 |
| 超范围 | 是否超出平台范围 | 改为拒绝 |
| 幻觉引用 | 引用的帖子是否真在检索结果中 | 删除虚假引用 |
| 完整性 | 是否遗漏关键步骤/信息 | 补充 |
| PII | 是否泄露用户 PII | 脱敏 |

## 三、反思实现

### 3.1 在线快速反思（V3）
- Final Answer 生成后，调一次 V3 反思：
  ```
  输入: query + 检索结果 + final_answer
  输出: {pass: bool, issues: [...], fixed_answer: "..."}
  ```
- pass=true → 输出原答案。
- pass=false → 输出 fixed_answer（若质量可接受）or 触发澄清。

### 3.2 触发条件
- **全量反思**成本高，仅对低 confidence 触发：
  - 意图分类 confidence < 0.7。
  - 检索 Top-1 分数 < 0.5。
  - 答案含「可能/也许/不确定」等模糊词。
- 高 confidence 直接输出，省反思。

### 3.3 离线深度反思（R1）
- 每日抽样 5% 答案用 R1 深度反思，结果入库。
- 用于发现系统性问题（某类 query 总幻觉）。

## 四、引用一致性校验（硬规则）

- 答案中每个 `[1]` `[2]` 引用编号必须对应 refs 列表。
- 引用的 post_id 必须在当轮 search_posts 工具返回的 data 中。
- 不一致直接判定反思失败。

## 五、反思失败的处理

- 反思失败且无法修复 → 输出保守答案：
  - 「根据检索，为您找到以下相关内容：」+ 帖子卡列表（不生成自然语言陈述）。
- 避免输出有问题的自然语言答案。

## 六、决策记录 (ADR)

### ADR-037: 反思条件触发
- **理由**：全量反思翻倍成本与延迟；低 confidence 触发性价比最高。
- **阈值**：可通过 A/B 调，初始 confidence < 0.7 触发。

### ADR-038: 引用一致性硬校验
- **理由**：幻觉引用是最伤信任的问题；硬规则比 LLM 反思更可靠。
- **实现**：正则提取答案中引用编号，与 refs 比对。
