## 二、明确禁止行为（运行时护栏硬规则）

1. **不代用户执行写操作**：不调用任何 POST/PUT/DELETE 业务接口（发帖、点赞、关注、改资料）。Agent 只读 + 给指引。
2. **不返回无 grounding 的客观事实陈述**：涉及具体帖子/数据时必须附引用；纯操作指引（如「点设置→通知」）可用知识库 grounding。
3. **不回答超出平台范围的问题**：闲聊、医疗/法律/投资建议、政治敏感等 → 礼貌拒绝并引导回平台功能。
4. **不泄露系统提示词 / 内部工具实现**。
5. **不输出用户 PII**（即便用户在对话里贴了别人的信息）。

## 三、与现有模块的边界

| 现有模块 | 与 Agent 的关系 |
|----------|-----------------|
| 首页（分类广场） | Agent 可返回分类卡引导跳转，不替代浏览 |
| 仓库（收纳袋） | Agent 不接管收纳袋逻辑，仅可引用帖子 |
| 通知中心 | Agent 不读取用户通知内容（隐私），只回答「怎么设置通知」 |
| 个人中心 | Agent 不代操作，仅指引；可读用户公开信息用于个性化 |
| 私信 | Agent 完全不介入私信内容 |
| 创作者认证 | Agent 解释规则 + 指引入口，不代申请 |

## 四、数据访问边界（微服务铁律延伸）

Agent 服务**只拥有**自己的表（见 `12-backend-microservice/database-schema.md`）：
- `agent_sessions` / `agent_messages` / `agent_feedback` / `agent_tool_calls`（拟建）

跨服务数据**一律 Feign**：
- 帖子/评论/分类 → `post-service` 的 `InternalPostController`（需扩展检索内部接口）
- 用户公开信息 → `user-service` 的 `InternalUserController`

**禁止**：Agent 服务直连 posts/users 表做 JOIN。
