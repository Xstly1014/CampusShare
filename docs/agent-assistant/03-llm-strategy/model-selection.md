## 三、选型决策

### 3.1 主生成模型：DeepSeek-V3 (deepseek-chat)
- **理由**：中文强、tool use 稳定、性价比极高（缓存命中后输入 ¥0.1/M）、国内访问无障碍、API 兼容 OpenAI 格式（SDK 通用）。
- **角色**：意图分类、查询改写、ReAct 主循环、最终答案生成。

### 3.2 复杂推理模型：DeepSeek-R1 (deepseek-reasoner)
- **理由**：进阶阶段用于「反思自校验」「复杂规划」「难例意图判定」，R1 推理链强。
- **注意**：R1 tool use 较弱，不直接进 ReAct 循环，只在「反思/规划」离线步骤用。

### 3.3 兜底模型：豆包 doubao-pro / Qwen-Max
- **理由**：DeepSeek 限流或不可用时秒级切换；豆包国内延迟最低。
- **切换条件**：主模型连续 3 次超时/5xx，或限流 429。

### 3.4 Embedding 模型：BGE-M3（智源）
- **理由**：中文检索 SOTA、支持多语言、1024 维、开源可自部署、支持稠密+稀疏+ColBERT 多模式。
- **API**：用硅基流动/智源官方 API；未来可自部署降本。
- **备选**：`bge-large-zh-v1.5`、`text-embedding-3-large`(OpenAI)。

### 3.5 重排模型（进阶）：bge-reranker-v2-m3
- **理由**：与 BGE-M3 同系列，中文重排效果好，cross-encoder 架构。

## 四、多模型分工策略

```
┌─────────────────────────────────────────────┐
│  意图分类(轻量)  → DeepSeek-V3 (低温度, JSON) │
│  查询改写        → DeepSeek-V3               │
│  ReAct 主循环    → DeepSeek-V3 (tool use)    │
│  最终答案生成    → DeepSeek-V3 (流式)         │
│  反思自校验(进阶)→ DeepSeek-R1               │
│  Embedding       → BGE-M3                    │
│  重排(进阶)      → bge-reranker-v2-m3        │
└─────────────────────────────────────────────┘
```

## 五、成本估算

假设：日均 1000 次问答，每次平均输入 2K tok + 输出 500 tok。

| 项 | 计算 | 日成本 |
|----|------|--------|
| 生成(DeepSeek-V3) | 1000 × (2K×¥1/M + 500×¥2/M) | ¥3 |
| 意图分类(单次 500 in/50 out) | 1000 × (500×¥1/M + 50×¥2/M) | ¥0.6 |
| Embedding(帖子+query) | query 1000×200tok + 增量帖子 | ¥0.3 |
| **合计** | | **≈ ¥4/天，¥120/月** |

单次问答 ≈ ¥0.004，远低于 ¥0.03 目标。预算充足，可承受更复杂的多步 Agent。

## 六、决策记录 (ADR)

### ADR-006: 主模型选 DeepSeek-V3
- 性价比、中文、tool use、国内可达四项综合最优。
- 风险：DeepSeek 偶发限流 → 用豆包兜底缓解。

### ADR-007: 推理模型 R1 仅用于离线反思
- R1 推理延迟高(数秒~十几秒)、tool use 弱，不适合在线 ReAct 循环。
- 用于发版前批量反思校验、难例人工辅助。

### ADR-008: Embedding 选 BGE-M3
- 中文检索质量、多模式(稠密/稀疏/ColBERT)、开源可自部署。
- 1024 维平衡质量与存储。
