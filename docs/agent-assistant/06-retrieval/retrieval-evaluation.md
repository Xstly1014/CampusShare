# 检索评估

> 状态: 草稿  
> 最后更新: 2026-06-30

## 一、评估指标

| 指标 | 定义 | 目标 |
|------|------|------|
| Recall@K | 前 K 召回含相关帖的比例 | Recall@20 ≥ 95% |
| Precision@5 | 前 5 中相关比例 | ≥ 70% |
| MRR | 平均倒数排名 | ≥ 0.6 |
| NDCG@5 | 考虑排序的归一化折损累计增益 | ≥ 0.7 |
| Top-5 命中率 | 前 5 含用户主观相关帖 | ≥ 85% |
| 检索延迟 P95 | - | ≤ 500ms |

## 二、黄金测试集

- 来源：`01-requirements/user-scenarios.md` 的 SEARCH 场景 + 人工标注。
- 每条含：
  - `query`：原 query
  - `relevant_post_ids`：人工判定相关的帖子 id 列表（含必相关 / 可能相关两级）
  - `slots_expected`：期望槽位
- 规模：初始 50 条，逐步扩到 200+。

## 三、评估流程

```
1. 跑黄金集所有 query
2. 记录每条 Top-20 召回结果
3. 计算 Recall@20, Precision@5, MRR, NDCG@5
4. 失败 case 分析（召回为空 / 相关帖排太低 / 槽位抽取错）
5. 改 prompt / 同义词表 / 重排权重
6. 重跑验证
```

## 四、A/B 实验

- 线上分流：10% 流量走实验组（新检索策略），90% 对照组。
- 指标：用户 👍 率、帖子卡点击率、会话深度。
- 显著性：t 检验 p < 0.05。

## 五、LLM-as-judge

- 用 DeepSeek-R1 对 (query, top5 结果) 打分：
  - 相关性（0-1）
  - 排序合理性（0-1）
  - 是否有幻觉引用（0/1）
- 用于离线批量评估，补充人工标注不足。

## 六、失败 case 归因

| 现象 | 可能原因 | 改进 |
|------|----------|------|
| 召回为空 | 槽位过严 / 同义词缺失 | 放宽过滤 + 补同义词 |
| 相关帖未召回 | 向量质量差 / 分块丢失语义 | 调向量化内容构造 |
| 相关帖召回但排低 | 重排未开 / 业务加权不当 | 开重排 / 调权重 |
| 槽位抽错 | LLM 分类 prompt 不清 | 补 Few-shot |
| 跨板块误召 | 向量语义漂移 | 加结构化过滤 |

## 七、决策记录 (ADR)

### ADR-030: 评估黄金集 + LLM-as-judge 双轨
- **理由**：人工标注贵且慢，LLM-as-judge 可批量；两者结合互补。
- **频率**：黄金集每次发版跑；LLM-as-judge 每日抽样跑。
