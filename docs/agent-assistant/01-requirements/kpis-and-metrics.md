# KPI 与度量指标

> 状态: 草稿  
> 最后更新: 2026-06-30

## 一、北极星指标

**用户问答 👍 率**：所有 Agent 回答中用户点 👍 的比例。  
目标：≥ 70%。这是答案质量的最直接信号，优于任何离线指标。

## 二、分层 KPI

### 2.1 检索层
| KPI | 定义 | 目标 | 采集 |
|-----|------|------|------|
| Top-5 命中率 | 黄金集 query 的 Top-5 含正确帖的比例 | ≥ 85% | 离线评估 |
| Recall@20 | 召回集含相关帖的比例 | ≥ 95% | 离线评估 |
| MRR | 平均倒数排名 | ≥ 0.6 | 离线评估 |
| 检索延迟 P95 | 单次混合检索耗时 | ≤ 500ms | trace |

### 2.2 生成层
| KPI | 定义 | 目标 | 采集 |
|-----|------|------|------|
| 👍 率 | 👍 / (👍+👎) | ≥ 70% | 在线反馈 |
| 无引用编造型回答率 | 抽样 audit 中无 grounding 的客观陈述比例 | < 2% | 人工 audit |
| 答案相关性 | 答案是否切题（LLM-as-judge） | ≥ 0.8 | 离线评估 |
| 超范围拒绝正确率 | 超范围问题正确拒绝比例 | ≥ 95% | 离线评估 |

### 2.3 性能层
| KPI | 目标 |
|-----|------|
| 首 token P95 | ≤ 1.5s |
| 完整回复 P95 | ≤ 6s |
| 工具调用 P95 | ≤ 800ms |
| 服务可用率 | ≥ 99% |

### 2.4 成本层
| KPI | 目标 |
|-----|------|
| 单次问答平均成本 | ≤ ¥0.03 |
| 月度 LLM 费用 | 可观测、可告警 |

### 2.5 采纳层
| KPI | 定义 |
|-----|------|
| 帖子卡点击率 | Agent 返回的帖子卡被点击的比例 |
| 跳转完成率 | 用户点击跳转后停留 > 5s 的比例 |
| 会话深度 | 平均每会话轮数 |

## 三、度量采集方式

1. **离线评估**：黄金测试集（见 `13-evaluation/test-datasets.md`）+ 自动化脚本，每次发版前跑。
2. **在线 trace**：每次问答全链路 trace（OTel），含意图、工具调用、检索、LLM 各段耗时。
3. **用户反馈**：👍/👎 + 文字反馈，入库。
4. **LLM-as-judge**：用更强模型对抽样答案打分（相关性、grounding、简洁性）。
5. **人工 audit**：每周抽样 50 条人工复核，重点查幻觉与超范围。

## 四、告警阈值

- 首 token P95 > 3s 持续 5min → 告警
- 服务错误率 > 1% 持续 2min → 告警
- 👍 率连续 3 天 < 50% → 告警（质量回退）
- 单日成本超预算 120% → 告警
