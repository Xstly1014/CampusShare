# 功能需求与非功能需求

> 状态: 草稿  
> 最后更新: 2026-06-30

## 一、功能需求（FR）

### FR-1 对话管理
- FR-1.1 创建会话、按会话维持多轮上下文（短期记忆窗口 ≥ 10 轮）。
- FR-1.2 流式输出（SSE），首 token 尽快返回。
- FR-1.3 支持用户主动「清空当前会话」。
- FR-1.4 会话历史持久化，用户重进可看历史（脱敏存储，见 `14-safety-and-guardrails`）。

### FR-2 意图理解与路由
- FR-2.1 识别 5 大意图：HOW_TO / SEARCH / NAVIGATE / CLARIFY / OUT_OF_SCOPE。
- FR-2.2 多轮上下文下做指代消解与意图修正。
- FR-2.3 查询改写（同义词、缩写、HyDE）。

### FR-3 知识问答
- FR-3.1 帮助中心知识库覆盖全部已上线功能。
- FR-3.2 回答附带操作步骤 + 跳转入口。
- FR-3.3 知识库可后台更新，无需重启服务。

### FR-4 帖子检索
- FR-4.1 跨学校、跨分类语义检索。
- FR-4.2 支持过滤：学校、分类、帖子类型（资源/讨论）、排序（最新/最热）。
- FR-4.3 混合检索（向量 + 关键词 + 结构化）。
- FR-4.4 重排序（进阶）。
- FR-4.5 召回为空时降级推荐相近板块。

### FR-5 工具调用
- FR-5.1 Agent 可调用只读工具集（见 `10-tools-and-apis`）。
- FR-5.2 工具调用过程对用户可见（「正在检索帖子…」）。
- FR-5.3 工具调用失败有降级（返回错误提示而非崩溃）。

### FR-6 引用与跳转
- FR-6.1 答案中帖子/分类/用户以可点击卡片呈现。
- FR-6.2 跳转复用现有前端路由。
- FR-6.3 引用编号与答案正文对应。

### FR-7 反馈
- FR-7.1 每条 Agent 回答下方有 👍/👎。
- FR-7.2 👎 可选填文字反馈。
- FR-7.3 反馈入库存档供评估。

### FR-8 安全
- FR-8.1 超范围问题礼貌拒绝（见 `14-safety-and-guardrails`）。
- FR-8.2 不代用户执行写操作。
- FR-8.3 不输出 PII。

## 二、非功能需求（NFR）

| 维度 | 指标 | 目标 |
|------|------|------|
| 性能 | 首 token P95 | ≤ 1.5s |
| 性能 | 完整回复 P95 | ≤ 6s |
| 性能 | 工具调用（单次）P95 | ≤ 800ms |
| 可用性 | 服务可用率 | ≥ 99% |
| 可用性 | 主 LLM 不可用时降级生效时间 | ≤ 5s |
| 扩展性 | 向量库可承载帖子数 | ≥ 100 万 |
| 扩展性 | 单节点 QPS | ≥ 20（检索类）/ ≥ 5（生成类） |
| 成本 | 单次问答平均 | ≤ ¥0.03 |
| 成本 | 月度 LLM 费用可观测 | 100% |
| 安全 | 无引用编造型回答率 | < 2% |
| 安全 | 超范围问题正确拒绝率 | ≥ 95% |
| 可观测 | 每次问答全链路 trace 覆盖率 | 100% |
| 兼容 | 移动端可用 | 100%（底部 NavBar） |
| 国际化 | 中文为主 | 是 |

## 三、KPI 与度量（详见 `01-requirements/kpis-and-metrics.md`）

2. 用户 👍 率
3. 首 token P95
4. 单次成本
5. 超范围拒绝正确率
