## 一、产品概述

### 1.1 产品定位

CampusShare 是一款面向高校学生的校园资源共享与社区交流 H5 应用，旨在打破校园间的信息壁垒，让优质学习资源和校园经验在全国高校间流动。

### 1.2 产品愿景

成为国内领先的高校资源共享社区，让每一位大学生都能便捷地获取来自全国名校的优质学习资源，同时通过社区交流促进跨校学术互动。

### 1.3 目标用户

- **核心用户**：全国高校本科生、研究生
- **内容生产者**：乐于分享学习资料、校园经验的学生
- **内容消费者**：寻找学习资料、校园攻略的学生

### 1.4 核心价值

| 价值维度 | 具体描述 |
|---------|---------|
| **资源价值** | 聚合多所高校的优质学习资源，降低获取门槛 |
| **社交价值** | 以校为单位的社区交流，认识同校/跨校同学 |
| **效率价值** | 精准搜索与筛选，快速找到所需资料 |
| **激励价值** | 创作者认证体系，让优质分享者获得回报 |

## 三、功能详细说明

### 3.1 认证模块

#### 3.1.1 用户注册

**功能描述**：支持手机号和邮箱两种注册方式

**业务规则**：
- 手机号注册：6位短信验证码（mock 环境下随机生成，Redis 存储5分钟）
- 邮箱注册：6位邮箱验证码（mock 环境下随机生成，Redis 存储5分钟）
- 密码要求：6-20位，需包含字母和数字
- 用户名：3-20位，支持中英文、数字、下划线
- 同一账号不可重复注册
- 注册成功后自动登录

**交互流程**：
```
选择注册类型 → 输入账号 → 获取验证码 → 输入验证码 → 设置用户名 → 设置密码 → 注册成功
```

#### 3.1.2 用户登录

**功能描述**：支持手机号/邮箱 + 密码登录

**业务规则**：
- 支持手机号或邮箱作为登录账号
- 密码错误5次锁定账号15分钟（待实现）
- 支持"记住密码"，有效期7天
- JWT Token 机制，Access Token 有效期2小时，Refresh Token 有效期7天

#### 3.1.3 忘记密码

**功能描述**：通过验证码重置密码

**业务规则**：
- 需输入注册时的手机号/邮箱
- 验证码校验通过后可设置新密码
- 新密码不能与原密码相同

### 3.2 首页模块

#### 3.2.1 学校列表

**功能描述**：以卡片网格形式展示所有高校

**展示规则**：
- 响应式布局：移动端2列 / 平板3列 / 桌面4列
- 每所学校展示：校徽、学校名称、资源数量
- 支持搜索过滤（按学校名称）
- 默认按资源数量降序排列

#### 3.2.2 搜索功能

**功能描述**：顶部搜索栏，实时过滤学校列表

**交互规则**：
- 输入即时过滤，防抖 300ms
- 支持学校名称模糊匹配
- 无结果时显示空状态提示

### 3.3 学校社区模块

#### 3.3.1 帖子列表

**功能描述**：展示当前学校下的所有帖子

**筛选与排序**：
- **类型筛选**：全部 / 资料 / 讨论
- **排序方式**：
  - 最新：按发布时间倒序
  - 最热：按收藏数倒序（类似 GitHub Star）
  - 活跃：按评论数倒序

**帖子类型**：
| 类型 | 标识颜色 | 核心元素 |
|------|---------|---------|
| 资料贴 | 蓝色 | 文件信息（格式/大小）、资源描述 |
| 讨论贴 | 橙色 | 正文内容摘要 |

#### 3.3.2 发布帖子

**功能描述**：用户可在学校社区内发布帖子

**类型差异**：
| 维度 | 资料贴 | 讨论贴 |
|------|--------|--------|
| 标题提示 | "请输入资源名称，例如：高等数学期末复习资料" | "请输入帖子标题，吸引大家来讨论" |
| 内容提示 | "简单介绍一下这份资源的内容、适用人群、参考价值等..." | "分享你的想法、问题或者经验，和大家一起讨论..." |
| 文件要求 | 必填 | 可选 |
| 字数限制 | 标题100字 / 描述2000字 | 标题100字 / 正文2000字 |

**支持的文件格式**：
- 文档：PDF、DOC、DOCX、XLS、XLSX、PPT、PPTX
- 文本：TXT、MD、HTML
- 压缩包：ZIP、RAR、7Z
- 图片：JPG、PNG、GIF
- 单文件大小限制：50MB

#### 3.3.3 帖子详情

**功能描述**：展示帖子完整内容及评论区

**资料贴详情**：
- 帖子标题、作者信息、发布时间
- 资源描述正文
- 文件卡片（文件名、格式、大小、下载按钮）
- 数据统计（浏览量、收藏数、评论数）
- 评论区

**讨论贴详情**：
- 帖子标题、作者信息、发布时间
- 正文内容
- 附件（如有）
- 数据统计
- 评论区

**评论功能**：
- ✅ 一级评论发布与展示（按时间正序）
- ✅ 评论发布后即时展示
- ✅ 发评论时帖子 comment_count 自动 +1
- ✅ 楼中楼回复（回复显示"用户名 回复 @被回复用户名"，按时间正序，楼内可再回复）
- ✅ 评论点赞
- ✅ 评论删除（评论作者或帖子作者可删）

### 3.4 个人主页模块

#### 3.4.1 个人资料

**可编辑项**：
- 头像：支持本地图片上传，上传成功后提示"头像修改成功"
- 用户名：3-20位，修改时检查重复
- 个人简介：最多200字
- 保存方式：编辑弹窗中修改后点击保存按钮生效

#### 3.4.2 认证创作者

**功能描述**：创作者认证申请入口（UI 占位，后续迭代实现）

**规划方向**：
- 官方审核通过的认证形式
- 认证创作者享有专属标识
- 激励奖励机制（积分、虚拟礼物等）
- 优质资源推荐加权

#### 3.4.3 我的互动

**入口按钮模式**（类似抖音，点击进入独立列表页）：
| 入口 | 说明 |
|-----|------|
| 我的帖子 | 展示用户自己发布的所有帖子，按发布时间倒序 |
| 我的回复 | 展示用户发表的所有评论，按评论时间倒序 |
| 浏览历史 | 按最近浏览时间倒序展示浏览过的帖子（同一帖子只保留最近一条记录） |
| 我的收藏 | 展示用户收藏的所有帖子，按收藏时间倒序 |
| 我的点赞 | 展示用户点赞过的所有帖子，按点赞时间倒序 |

#### 3.4.4 作者数据统计

**可点击统计**（个人主页顶部，6项分两组）：

**第一组：成就展示（点击弹出风格化弹窗）**
| 统计项 | 说明 | 弹窗文案 |
|--------|------|---------|
| 总浏览 | 我发布的所有帖子被浏览的总次数 | "你迄今为止获得了xxx个总浏览哦~" |
| 获赞 | 我发布的所有帖子被点赞的总次数 | "你迄今为止获得了xxx个赞哦~" |
| 被收藏 | 我发布的所有帖子被收藏的总次数 | "你迄今为止获得了xxx次收藏哦~" |

**弹窗交互**：
- 点击统计项弹出全屏遮罩（背景变暗 bg-black/50）
- 弹窗卡片采用蓝紫渐变（from-blue-500 to-purple-600）
- 展示大号数字 + 文案 + 关闭按钮
- 点击遮罩或关闭按钮关闭弹窗

**第二组：社交关系（点击跳转列表页）**
| 统计项 | 说明 | 跳转页面 |
|--------|------|---------|
| 关注 | 我关注了多少用户 | `/profile/following` |
| 粉丝 | 有多少用户关注了我 | `/profile/followers` |
| 互关 | 互相关注的用户数 | `/profile/mutual` |

#### 3.4.5 用户社交

**关注/取消关注**：
- 在用户主页（`/user/:userId`）可关注/取消关注其他用户
- 关注关系存储于 `follows` 表（follower_id → following_id）
- 互关 = 双方互相关注

**关注/粉丝/互关列表**：
- 列表页展示用户头像、用户名、简介
- 点击用户项跳转到该用户主页
- 每项右侧提供私信快捷按钮

**查看用户主页**：
- 展示用户资料、统计数据、帖子/收藏/点赞/浏览 Tab
- 提供关注按钮和私信按钮
- 支持复制用户 ID

#### 3.4.6 私信模块

**功能描述**：用户间一对一私信交流

**会话列表**（`/messages`）：
- 展示所有私信会话，按最近消息时间倒序
- 每条显示对方头像、用户名、最近消息摘要、时间

**聊天页面**（`/messages/:userId`）：
- 展示与指定用户的完整对话记录
- 消息气泡：自己发的靠右蓝色，对方发的靠左白色
- 底部输入框 + 发送按钮

**发送限制规则**：
- 若对方**未关注你**且**未回复过你** → 你只能发送**一条**消息
- 对方关注了你（单向即可）→ 可自由发送
- 对方回复过你（一条即可）→ 可自由发送
- 互相关注**不是**自由发送的必要条件，互相回复一条消息即可
- 达到限制时输入框禁用，提示"对方未关注你且未回复，等待对方回复后可继续发送"

### 3.5 资源仓库模块（部分实现）

**规划功能**：
- 我上传的资源：用户自己发布的所有资料贴
- 我下载的资源：下载过的文件记录

## 四、非功能需求

### 4.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 首屏加载时间 | < 2s（4G 网络） |
| 接口响应时间 | P95 < 500ms |
| 列表滚动帧率 | 60fps |
| 并发用户数 | 支持 1000 QPS |

### 4.2 安全要求

- 密码使用 BCrypt 加密存储，强度 10
- JWT Token 防篡改
- SQL 注入防护（MyBatis Plus 参数化查询）
- XSS 防护（前端输出转义）
- CSRF 防护（Token 机制）
- 文件上传类型校验，防止恶意文件

### 4.3 兼容性要求

- 移动端：iOS 12+ / Android 7+
- 浏览器：Chrome 80+、Safari 13+、微信内置浏览器
- 分辨率：320px - 1920px 自适应

### 4.4 可用性要求

- 系统可用性：99.9%
- 数据备份：每日全量备份 + 实时 binlog
- 故障恢复：RPO < 1min，RTO < 30min
